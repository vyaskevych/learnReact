Додаток Валютний інформер

Крок 1. Отримання даних

Перейдіть на сторінку API Національного банку України (https://bank.gov.ua/ua/open-data/api-dev). Огляньте розділ Офіційний курс гривні до іноземних валют та банківських металів.

Перейдіть за посиланням Курс на поточну дату: JSON. З'ясуйте структуру відповіді.

! Занотуйте у звіт адресу запиту курсу на поточну дату.

! Запишіть у звіт структуру відповіді. З чого складається, які поля присутні (назва та їх призначення) 

Відкрийте у браузері інструменти розробника (зазвичай F12) перейдіть на вкладку Console. Напишіть код який надішле запит на API НБУ (за допомогою методу fetch()) та виведе на консоль курс долара США.

Структура запиту:

fetch(url)
    .then(response=>response.json())
    .then(…)
    .catch(err=>console.error(err))    

! Занотуйте у звіт код та відповідь.

Крок 2. Структура додатку

Продумайте структуру додатку. В основному блоці для відображення списку курси валют на сьогодні, вивести поточну дату у місцевому форматі.

! Занотуйте у звіт код та пояснення к коду отримання поточної дати.

Крок 3. Отримання даних у додаток

Скористайтеся кодом з кроку 1 та збережіть отримані дані. Подумайте де краще розмістити код зі кроку 1, і як його викликати.  

! Занотуйте у звіт варіанти де саме можна розмістити код для отримання даних з API, та поясніть свій вибір. 

Крок 4. Відображення даних

Створіть функцію, яка виведіть у вигляді списку назву валюти та її курс. Не забувайте про оформлення додатку! Додайте слухач подій, який при кліку на вибрану валюту буде здійснювати новий запит для отримання курсу обраної валюти за певний період часу. Обробник цієї події буде зроблено пізніше.

! Занотуйте у звіт як саме можна підключити обробники подій, та поясніть свій вибір.

Крок 5. Отримання даних

Перейдіть на сторінку API Національного банку України (https://bank.gov.ua/ua/open-data/api-dev). Огляньте розділ Офіційний курс гривні до іноземних валют та банківських металів. 

Перейдіть за посиланням Курс на дату по валюті: JSON. З'ясуйте структуру відповіді.

! Занотуйте у звіт адресу запиту курсу по валюті на задану дату.

! Запишіть у звіт структуру відповіді. З чого складається, які поля присутні (назва та їх призначення) 

Відкрийте у браузері інструменти розробника (зазвичай F12) перейдіть на вкладку Console. Напишіть код який надішле запит на API НБУ (за допомогою методу fetch()) та виведе на консоль курс Євро на позавчора.

Структура запиту:

fetch(url)
    .then(response=>response.json())

! Занотуйте у звіт код та відповідь.

Крок 6. Рефакторинг коду

Винесіть запит до API в окрему асинхронну функцію, що приймає url адресу та повертає об'єкт JS. Додайте виклик цієї функції. Не забувайте про await. Переконайтеся, що додаток працює належним чином. 

! Занотуйте у звіт що таке async/await.

Крок 7. Перехід на екран курсу обраної валюти за тиждень

Напишіть функцію обробник події клік на валюту. На першому етапі відобразіть у блоці для відображення курсу обраної валюти за тиждень – код обраної валюти.

Крок 8. Курс валюти за тиждень

Вручну сформувати дати в необхідному для API форматі. За допомогою функції з кроку 6 надіслати запити та отримати відповідь. Вивести отримані дані на екран. Для відправлення запитів (7 штук) можна скористатися методом promise.all (https://learn.javascript.ru/promise-api)

Крок 9. Стилізація

Зробить Ваш додаток охайним та унікальним.

Крок 10. Автоматичне отримання дат

Напишіть JS функцію (поки що окремо від додатку) яка зчитує поточну дату перетворює її на формат необхідний для API НБУ (задається у форматі YYYYMMDD, де YYYY - рік, MM - місяць, DD - день) та записує її у масив. Далі від поточної дати віднімає один день і знову перетворює цю нову дату на необхідний формат та додає її у масив. Ці дії повторюються задану зовні кількість разів (днів). Отриманий масив повернути із функції. Функцію запустить у консолі. Корисні посилання:

  • https://learn.javascript.ru/datetime
  • https://stackoverflow.com/questions/1296358/subtract-days-from-a-date-in-javascript

! Занотуйте у звіт код та результат роботи функції.

Крок 11. Завершальний

Додайте функцію, що створена на кроці 10 до додатку. Зформуйте запити до API використовуючи дати, що отримані з функції. Змініть кількість днів, перевірте, що все працює належним чином. 

Під час запуску додатку іноді можна побачити, що дати розміщені випадково. Це трапляється тому що запити в promise.all асинхронні. Виправте це шляхом впорядкування (сортування) отриманих даних за датою (від найсвіжішої до найстарішої).

! Запишіть у звіт версію додатку
